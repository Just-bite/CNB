# Token Ring Network Simulator

Интерактивный эмулятор локальной сети с топологией «Кольцо» и маркерным доступом (протокол Token Ring). Проект визуализирует передачу данных, работу управляющего маркера и механизмы восстановления сети при сбоях.

## 🚀 Основные возможности
* **Визуализация топологии:** Динамическое построение кольца станций (от 2 до 12 узлов) с использованием `QGraphicsScene`.
* **Эмуляция протокола:** 
    * Реализация кадра Token Ring (SD, AC, FC, DA, SA, Data, FCS, ED, FS).
    * Механизм захвата свободного маркера и установка битов состояния кадра (Address Recognized, Frame Copied).
    * Поддержка роли **Активного монитора**: контроль "здоровья" кольца и регенерация маркера.
* **Стресс-тестирование:**
    * Отключение станций.
    * Имитация потери маркера  с последующим восстановлением монитором.
    * Повреждение данных и изменение адреса назначения в процессе передачи.
* **Cyberpunk UI:** Стилизованный интерфейс в темных тонах с использованием QSS и встроенным логгером событий.

## Технологический стек
* **Язык:** C++11/17
* **Фреймворк:** Qt 5.14.2+ 

## Структура проекта
* `src/` — реализация логики.
* `include/` — заголовочные файлы и определения состояний.
* `resources/` — графические ресурсы (иконки станций, маркера).
* `mainwindow.ui` — описание интерфейса.

## 🔧 Сборка и запуск

### Требования
* Qt Creator 4.11.1+
* Qt 5.14.2+ 
* Компилятор: MinGW / MSVC(2017+)

### Инструкция
1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Just-bite/token-ring-simulator.git
   ```
2. Откройте файл `TokenRing.pro` в Qt Creator.
3. Убедитесь, что ресурсы (иконки) находятся в папке `resources/`.
4. Выполните **Clean** -> **Run qmake** -> **Build**.
5. Запустите проект.

## Как пользоваться эмулятором
1. **Настройка:** Введите количество станций и нажмите **Start Simulation**.
2. **Передача:** Выберите адрес отправителя (Sender), получателя (Receiver), введите текст и нажмите **Send**.
3. **Мониторинг:** Наведите курсор на станцию, чтобы увидеть её текущее состояние, очередь сообщений и роль в сети в виде ToolTip.
4. **Ошибки:** Используйте кнопку **Modulate error**, чтобы проверить, как сеть справится с потерей маркера или выходом узла из строя.
